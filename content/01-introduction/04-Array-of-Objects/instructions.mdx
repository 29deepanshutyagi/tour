# Array of Objects



Now let's add `skills` field. Which is an array of objects. Each object in the array will have two properties `name` and `level`.

- The `name` property will be a string value.
- The `level` property will have only three possible values: `beginner`, `intermediate`, and `advanced`.



```json highlightLineStart={4} highlightLineEnd={13}
{
  "name": "John Doe",
  "age": 25,
  "skills": [
    {
      "name": "JavaScript",
      "level":"beginner"
    },
    {
      "name": "React",
      "level":"intermediate"
    }
  ]
}
```







## Schema for the Array of Objects


In _JSON Schema_ you can use the `items` keyword for defining the type (or _subschema_) of items in the array.

in this case, you can use the `items` keyword with the `type` set to `object`, and to constrain the values of `skills.level` you can use the `enum` keyword
```json
{
  "skills": {
    "type": "array",
    "items": {
      "type": "object",
      "properties": {
        "name": {"type": "string"},
        "level": { "enum": ["beginner", "intermediate", "advanced"] }
      }
    }
  }
}
```
