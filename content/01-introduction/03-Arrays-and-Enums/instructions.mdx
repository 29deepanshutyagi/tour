# Enumerated Values

**How do you constrain a property to have a fixed set of values?** Enumerated values are used to solve this problem.


Consider we add a field `hobbies` to the document. The `hobbies` field can have only three possible values: `reading`, `writing`, and `painting`.


```json highlightLineStart={4}
{
  "name": "John Doe",
  "age": 25,
  "hobbies": "reading"
}
```

To define the enumerated values in _JSON Schema_, use the `enum` keyword. **Example:**

```json 
{
  "hobbies": {
    "type": "string",
    "enum": ["reading", "writing", "painting"]
  }
}
```

# Arrays

We will convert the hobbies field to an array of strings. 
  
  ```json highlightLineStart={4}
{
  "name": "John Doe",
  "age": 25,
  "hobbies": ["reading", "writing"]
}
```


In _JSON Schema_, defining an array is similar to defining an object. Use the `type` keyword with the value `array` to define an array. with the `items` keyword to define the type (or _subschema_) of items in the array. 

please see the below example, where we have declared hobbies as an array of strings.


```json
{
  "hobbies": {
    "type": "array",
    "items": {
      "type": "string"
    }
  }
}
```

In this step, we will define an **array** property `hobbies`, which will **consist of zero or more predefined values**. in our case, the array will consist of the values from `reading`, `writing`, and `painting`. any other value will be invalid.

### Schema Defination

```json
 {
  "hobbies": {
    "type": "array",
    "items": {
      "type": "string",
      "enum": ["reading", "writing", "painting"]
    }
  }
 }
```
